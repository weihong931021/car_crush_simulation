# 錯誤修復需求文檔

## 介紹

修復與監視器方向垂直行駛的機車軌跡跳動問題。當機車垂直於監視器方向行駛時，其被偵測到的位置會出現左右跳動現象，導致與監視器的平行距離大幅變動，使得在地圖上的軌跡不是平滑的直線。此問題需要透過在inference_config.yaml中新增針對機車的kinematics參數配置來解決，參考現有的car_heading_smooth模式進行優化。

## 錯誤分析

### 當前行為（缺陷）

1.1 當機車與監視器方向垂直行駛時，系統偵測到的位置會出現左右跳動現象
1.2 當機車垂直行駛時，系統計算的與監視器平行距離會大幅變動
1.3 當機車垂直行駛時，系統在地圖上顯示的軌跡不是平滑的直線而是鋸齒狀

### 預期行為（正確）

2.1 當機車與監視器方向垂直行駛時，系統應該偵測到平滑穩定的位置
2.2 當機車垂直行駛時，系統應該計算出穩定的與監視器平行距離
2.3 當機車垂直行駛時，系統應該在地圖上顯示平滑的直線軌跡

### 不變行為（回歸預防）

3.1 當機車與監視器方向平行行駛時，系統應該繼續維持現有的軌跡平滑度
3.2 當機車以其他角度行駛時，系統應該繼續維持現有的軌跡品質
3.3 當汽車行駛時，系統應該繼續維持現有car_heading_smooth模式的效果
3.4 當其他車輛類型行駛時，系統應該繼續維持現有的軌跡處理效果
